# Data provenance (`./data`)

This artifact regenerates every table and figure from source. The data here fall into
three groups by who holds the rights and whether the file is shipped.

## 1. Included — author-derived aggregate tables

Small county×year aggregate tables compiled by the authors from public statistics. These are
the authors' own derived products and are shipped with the repository.

| File | Role |
|---|---|
| `F_SI_C_Rates_Cleaned.csv` | NYMTC model input: county×year crash/killed/serious counts and rates per 100M VMT |
| `nymtc_safety_panel.csv` | cleaned 60-row NYMTC panel (also regenerated by `src/02`) |
| `njtpa_panel.csv` | NJTPA replication input (G=13): crashes, killed (K), serious (A), NJDOT/HPMS VMT |
| `capital_district_rates.csv` | Capital District replication input (G=4), same schema as the rate file |
| `Contributing_Factors_corrected.csv` | corrected ITSMR contributing-factor counts (county×year×factor, all severities); input to `src/17` covariate extension (§4.5) |

Note on exposure: county×year **aggregate** VMT is implied by the published rates
(`Cract / Crart`). That coarse aggregate is the authors' derived figure; it is distinct from
the detailed NYSDOT functional-class DVMT workbook in §3, which is *not* redistributed here.

Underlying sources (all public). The NYMTC crash, fatality (K), and serious-injury (A) counts in
these tables are compiled from the ITSMR Traffic Safety Statistical Repository
(<https://www.itsmr.org/tssr>); the same regional crash data are also downloadable from the NYMTC
Crash Data Explorer 2019–2024 (<https://crashdata-nymtc.hub.arcgis.com/>). NYMTC county VMT is the
NYSDOT DVMT series (§3). The NJTPA panel is built from public NJDOT crash records and the NJDOT/HPMS
county VMT tabulation. The national FARS/FHWA benchmark series is embedded in `src/11` (§4).

## 2. Included — ITSMR (publicly accessible)

Day×Time×severity and contributing-factor tables from the Institute for Traffic Safety
Management and Research (ITSMR) Traffic Safety Statistical Repository, which is publicly
accessible. Shipped for exact reproducibility.

| File | Role | SHA-256 |
|---|---|---|
| `01_0108_Day_of_Week_Time_of_Day.xlsx` | all-crash Day×Time×severity (truck count-on-count demo; `src/09`, `src/12`) | `16cb490c94feff6afa719ec78af24b20347434c91fed723962a14368fbadceea` |
| `05_0506_Day_of_Week_Time_of_Day.xlsx` | large-truck Day×Time×severity (same) | `9e871842ee0f2ff356d42a1026eb973af868e89c25ed9a813ed22f5a3c07fae1` |
| `Contributing_Factors.xlsx`, `Crashes_With_{Contributing,Human,Environmental,Vehicular}_Factors.xlsx` | factor tables backing the paper's associational §5.3 | — |

**Correction (2026 refresh).** The legacy `*_Factors.xlsx` exports under-counted contributing-factor
totals by 1--2 orders of magnitude (e.g., Suffolk-2019 driver distraction read 444 versus the correct
10,614) and are retained only as a deprecated record. The covariate extension (`src/17`, manuscript
§4.5) reads the corrected `Contributing_Factors_corrected.csv` instead (all 10 counties × 2019--2024,
all-severity factor presence, no suppressed cells on the modeled factors). The remaining `*_Factors.xlsx`
files support only the descriptive §5.3 narrative and may be pruned without affecting any output.

## 3. NOT included — NYSDOT DVMT workbook (user-supplied)

`Draft_DVMT_and_Length_by_County_2016-2024_UAC_Update_1_.xlsx` — the New York State DOT (NYSDOT)
daily-VMT (DVMT) × roadway-length series, by functional class and urban area. **This third-party file is
not redistributed here.** It is needed only by the provenance check (`src/01`), and that step
runs without it.

- **What it is for.** `src/01` reconstructs annual county VMT from this workbook and confirms it
  equals the VMT implied by `F_SI_C_Rates_Cleaned.csv` to ratio = 1.0000 across all 60
  county-years — i.e., that the SPF offset traces to the official exposure series.
- **Recorded result (re-verifiable).** `ratio min=1.00000 max=1.00000 mean=1.00000` over 60
  county-years.
- **Version used.** SHA-256 `39be5ed071ef14c0c9066f60b6c86b2df0469161a556e01623db636f275c1edf`
  (lets anyone holding the same workbook confirm they have the version these results were produced from).
- **How to obtain.** NYSDOT's DVMT series is public: the statewide table is on New York Open Data
  ("Daily Vehicle Miles of Travel: Beginning 1985", dataset `fsr4-j7q5`), and the county-level DVMT
  appears in NYSDOT's annual Highway Mileage Reports (Highway Data Services Repository, dot.ny.gov).
  The analysis uses overall (not vehicle-class) county VMT, 2019--2024. The specific workbook here is
  a pre-publication draft cut and is not redistributed.
- **How to re-verify locally.** Place the workbook at
  `data/Draft_DVMT_and_Length_by_County_2016-2024_UAC_Update_1_.xlsx` (it is `.gitignore`d, so
  your local copy is never committed) and run `python src/01_verify_vmt_provenance.py`. With the
  file present, `src/01` runs the full live reconstruction and asserts the identity; without it,
  `src/01` reports the recorded result above and exits 0.

## 4. National benchmark (embedded, no file)

The national FARS / FHWA rate-ratio series used by `src/11` is embedded directly in that
script; there is no separate data file. Source: NHTSA FARS and FHWA travel-volume series.

## Swapping in your own agency data

To reproduce the method on a different region, replace the §1 CSVs with your own county×year
aggregates in the same schema (columns documented in `DOCS.md` §5) and, optionally, supply your
own exposure workbook for `src/01`. No third-party raw file is required to run the pipeline.
