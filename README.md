# Reproducible exposure-normalized safety-performance analysis (NYMTC, 2019–2024)

A dependency-light (numpy/scipy/pandas/openpyxl/matplotlib) pipeline that
regenerates every table and figure in the paper *From Counts to Rates: A
Reproducible Exposure-Normalized Safety-Performance Method for Small-Area Crash
Analysis*, from source.

## Quick start
```
pip install -r requirements.txt
python run_all.py          # reads ./data, writes ./outputs
```

## What the paper shows
1. **Method (contribution).** A safety-performance-function offset GLM
   (count ~ Poisson QMLE, `offset = log(VMT)`, county + year fixed effects) with
   few-cluster inference by a wild cluster bootstrap (G = 10), whose confidence
   intervals are obtained by inverting the restricted test so interval and test
   agree exactly.
2. **Diagnosis (contribution).** A count-on-count fixed-effects specification
   manufactures a spurious ~3.9× truck "lethality multiplier" on the same data
   whose honest conditional ratio is ~1.9× — reproducible yet not a valid
   estimand (**reproduction ≠ validation**).
3. **Result (correctness check).** The documented pandemic-era divergence —
   flat-to-falling crash rate, fatality rate elevated in four of five
   post-baseline years (peak 1.33 in 2021; 1.20 in 2024) — recovered cleanly by
   the right method.
4. **Replication (external validity).** The same estimator on the thirteen-county
   NJTPA region (G = 13) reproduces the divergence with real few-cluster power —
   crash rate down (0.86/0.90, *p* < 0.05) and fatality rate up (1.27, *p* < 0.05)
   in the pandemic years — tracking the national series at r = 0.78.
5. **Robustness.** The recovered year effects survive ±15% random and ±10%
   systematic VMT-offset error (the 2021 peak holds at 1.33, 2024 at 1.20).

## Provenance & inference notes
- Exposure: county VMT traces to the NYSDOT DVMT workbook (sum over functional
  classes/urban areas, ×days/year) and matches the rate file's implied VMT to
  ratio 1.0000 across all 60 county-years (step 01). That workbook is third-party
  and not redistributed here; step 01 records the verified result and re-runs the
  full check if you supply the workbook (see `data/README.md`). No vehicle-class split
  exists → no truck VMT, so the truck/non-truck split appears **only** in the count-on-count
  counter-demonstration (step 06); the paper forms no exposure-normalized truck risk
  rate and makes no causal claim about trucks.
- Point estimates use Poisson QMLE; overdispersion (Pearson up to ~156) and
  county clustering are carried by the wild cluster bootstrap, not a likelihood.
- With G = 10 the 2¹⁰ = 1,024 Rademacher sign vectors are enumerated exactly
  (no simulation error).

## Tests
A dependency-light test suite guards the core claims (run after `run_all.py`):
```
python tests/run_tests.py
```
- `test_enumeration.py` — the exact 2^10 = 1,024 wild-bootstrap sign-vector enumeration is complete, unique, and closed under negation.
- `test_provenance.py` — the provenance script reports the VMT identity (ratio ≈ 1.0000) and exits 0, whether reconstructing from the user-supplied DVMT workbook (live) or reporting the recorded result (documented).
- `test_bootstrap_ci.py` — grid-inverted bootstrap CI/test consistency, plus a regression check on the 2021 fatality inference (IRR ≈ 1.33, CI ≈ [1.14, 1.58]).
- `test_simulation_instability.py` — under a known-truth rare-event DGP, the conditional estimator is ~unbiased and tight while the count-on-count slope ratio is wide and sign-unstable.

## Reproducibility & engineering
The analysis is built to re-run deterministically on new data and to be verified
automatically rather than on trust:

- **Pinned environment.** `requirements.txt` gives loose lower bounds; `requirements.lock.txt`
  freezes the exact versions under which the suite passes (CPython 3.12 here; CI also runs 3.11).
- **Container.** `docker build -t nymtc-spf-repro .` builds the pinned environment and runs the
  test suite as its **final build step** — a failing build means a broken reproducibility claim.
  `docker run --rm nymtc-spf-repro` regenerates every output into `./outputs`.
- **Continuous integration.** `.github/workflows/ci.yml` installs the pinned environment and runs
  `python tests/run_tests.py` on every push/PR across Python 3.11 and 3.12, plus a Docker-build job
  and an informational `ruff` lint.
- **Determinism.** Few-cluster inference enumerates all `2^G` Rademacher sign vectors exactly at
  small `G` (no simulation error); beyond `G ≳ 25` it falls back to seeded sampled draws. Every
  stochastic step is seeded, so re-runs are bit-for-bit reproducible.

## Tests (`./tests`)
`python tests/run_tests.py` (no pytest needed) asserts: the exact `2^10 = 1,024` sign-vector
enumeration; the VMT exposure-provenance identity (ratio 1.0000 across all 60 county–years);
grid-inverted confidence-interval/test consistency; and the simulation's identification property
(the count-on-count multiplier is unidentified while the conditional rate sits at the truth).
